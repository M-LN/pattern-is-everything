01 — Foundations

Linear Regression

Supervised

// Fitting a line through data — the simplest predictive model

Linear regression finds the best-fit line through data. Given input x, we predict ŷ by learning weight w (slope) and bias b (intercept) that minimise prediction error.

ŷ = w · x + b
ŷ = prediction  |  w = weight  |  x = input  |  b = bias
MSE = (1/n) · Σ (yᵢ − ŷᵢ)²
MSE = Mean Squared Error loss  |  minimised by gradient descent

OLS Solution: For simple linear regression, the optimal weights have a closed-form: w = Σ(xᵢ−x̄)(yᵢ−ȳ) / Σ(xᵢ−x̄)². Gradient descent finds the same answer iteratively.
02 — Foundations

Gradient Descent

Optimization

// Rolling downhill on the loss landscape to find optimal weights

Gradient Descent is how models learn. We start at random weights and take small steps in the direction of steepest descent — opposite the gradient — to reach minimum loss.

w := w − α · ∂L/∂w
α = learning rate  |  ∂L/∂w = gradient (slope of loss w.r.t. weight)

Three variants differ in how many samples are used per update:

VariantSamples/stepProsCons
Batch GDAll nStable, exact gradientSlow per step
Stochastic GD1Fast, noisy escapes minimaHigh variance
Mini-batch GD32–256Best of both worldsBatch size is a hyperparameter

Learning rate: Too large → overshoot and diverge. Too small → extremely slow convergence. Learning rate warmup + decay schedulers (topic 10) solve this in practice.

Bias-Variance Tradeoff

Theory

// The fundamental tension between underfitting and overfitting

Every model makes two types of errors: Bias — systematic error from wrong assumptions (underfitting). Variance — sensitivity to noise in training data (overfitting). You can't minimize both simultaneously.

E[(y−ŷ)²] = Bias² + Variance + Irreducible Noise
Total expected error decomposes into these three independent terms

05 — Training

Loss Functions

Optimization

// Measuring how wrong the model is — the objective being minimised

The loss function defines what the model is optimising. Different tasks need different loss functions. Getting this wrong is one of the most common ML mistakes.

MSE = (1/n)·Σ(y−ŷ)²
Regression — squared penalty, sensitive to outliers
MAE = (1/n)·Σ|y−ŷ|
Robust regression — linear penalty, outlier-resistant
CE = −Σ yᵢ·log(ŷᵢ)
Classification — penalises confident wrong predictions exponentially
BCE = −[y·log(ŷ) + (1−y)·log(1−ŷ)]
Binary classification — cross-entropy for single output

06 — Training

Backpropagation

Algorithm

// Chain rule applied through a network — how every weight gets its gradient

Backprop answers: "how much did each weight contribute to the error?" It uses the chain rule to propagate gradients backward from the loss to every parameter.

∂L/∂w₁ = ∂L/∂ŷ · ∂ŷ/∂h · ∂h/∂w₁
Chain Rule: multiply local gradients along the path from loss back to each weight
// Forward pass → loss → backward pass → weight update
Click Animate
Vanishing gradients: In deep networks, multiplying many small numbers (sigmoid derivatives ≤ 0.25) makes early-layer gradients near zero. Solutions: ReLU activations, batch norm, residual connections.
07 — Training

Optimizers

Optimization

// Beyond vanilla SGD — momentum, adaptive learning rates, Adam

Modern optimizers improve on vanilla gradient descent by adding momentum (using past gradients) and adaptive rates (different step size per parameter).

SGD+Momentum: v := βv − α·∇L    w := w + v
β = momentum (typically 0.9) — accumulates velocity across steps
RMSProp: s := ρs + (1−ρ)·(∇L)²    w := w − α·∇L/√(s+ε)
Divides by RMS of recent gradients — large gradients get smaller steps
Adam: m̂ = m/(1−β₁ᵗ)    v̂ = v/(1−β₂ᵗ)    w := w − α·m̂/√(v̂+ε)
Adam = momentum + RMSProp + bias correction. Default: β₁=0.9, β₂=0.999, ε=1e-8
// Optimizer comparison on a saddle point surface
# PyTorch optimizers
optim = torch.optim.SGD(model.parameters(), lr=0.01, momentum=0.9)
optim = torch.optim.RMSprop(model.parameters(), lr=1e-3, alpha=0.99)
optim = torch.optim.Adam(model.parameters(), lr=1e-3, betas=(0.9, 0.999))
optim = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=0.01)  # Adam + L2
AdamW is the default choice for most modern models. It decouples weight decay from the gradient update, fixing a subtle bug in Adam's L2 regularization.
08 — Training

Regularization

Generalization

// Preventing overfitting by constraining model complexity

Regularization adds a penalty for complexity to the loss, discouraging the model from memorising noise and improving generalisation to unseen data.

L_total = L_data + λ·||w||²₂    (L2 / Ridge)
λ = regularization strength  |  drives weights toward zero but not exactly zero
L_total = L_data + λ·||w||₁    (L1 / Lasso)
L1 produces sparse weights — many exactly zero (implicit feature selection)
Dropout: h̃ = h ⊙ mask/p    mask ~ Bernoulli(p)
Each neuron zeroed with prob (1−p) during training; scaled by 1/p to preserve expected value
// L1 vs L2 penalty contours — see how they constrain weights differently
λ strength1.0
# L2 via weight_decay in optimizer
optim = torch.optim.AdamW(model.parameters(), weight_decay=1e-4)

# Dropout layer
self.dropout = nn.Dropout(p=0.3)   # drop 30% of neurons
x = self.dropout(x)                 # only active during training

# Automatically disabled at test time:
model.eval()   # disables dropout
model.train()  # re-enables dropout
09 — Training

Batch Normalization

Stabilization

// Normalizing activations to keep training stable and fast

Batch Normalization normalises each layer's activations to have zero mean and unit variance across the mini-batch, then scales/shifts with learned parameters γ and β.

μ_B = (1/m)·Σxᵢ    σ²_B = (1/m)·Σ(xᵢ−μ_B)²
Compute batch mean and variance
x̂ᵢ = (xᵢ − μ_B) / √(σ²_B + ε)
Normalise: ε = small constant for numerical stability (1e-5)
yᵢ = γ · x̂ᵢ + β
γ = learned scale  |  β = learned shift — restores representational power
// Effect of batch norm on activation distributions across layers
Layer Norm vs Batch Norm: BatchNorm normalises over the batch dimension — problematic for small batches and transformers. LayerNorm normalises over the feature dimension and is the standard in transformers.
self.bn = nn.BatchNorm1d(num_features=256)  # after linear
self.ln = nn.LayerNorm(normalized_shape=512) # after attention (transformers)
10 — Training

LR Scheduling

Optimization

// Adjusting the learning rate over training for better convergence

A fixed learning rate is rarely optimal. Starting too high causes instability; finishing too high prevents convergence. Schedulers adjust α during training to get the best of both.

Cosine Decay: αₜ = αₘᵢₙ + ½(αₘₐₓ−αₘᵢₙ)(1 + cos(πt/T))
T = total steps  |  smoothly decays from αₘₐₓ to αₘᵢₙ
Warmup: α = αₘₐₓ · (t/t_warmup)   for t < t_warmup
Linear ramp-up prevents large gradient updates from poorly-initialised weights
// Learning rate schedules — click to compare
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(optim, T_max=100)
scheduler = torch.optim.lr_scheduler.OneCycleLR(optim, max_lr=0.01, total_steps=1000)
scheduler = torch.optim.lr_scheduler.StepLR(optim, step_size=30, gamma=0.1)

# call after each epoch:
scheduler.step()
11 — Core Math

Softmax & Probabilities

Classification

// Converting raw scores into a probability distribution

Softmax maps a vector of real-valued logits to probabilities that sum to 1. It amplifies the largest logit, making the winner more decisive.

σ(zᵢ) = e^zᵢ / Σⱼ e^zⱼ
zᵢ = logit for class i  |  numerator = exponentiated score  |  denominator = normalisation
// Adjust logits — watch probabilities redistribute
Temperature scaling: σ(z/T). T<1 → sharper (more confident). T>1 → softer (more uniform). Used in knowledge distillation and language model sampling.
12 — Core Math

MLE & Gaussian Distributions

Probability

// Maximum Likelihood Estimation — why MSE and Cross-Entropy exist

MLE asks: what parameters make the observed data most probable? Most ML training objectives are secretly MLE under a particular assumed distribution.

θ* = argmax_θ Σ log p(xᵢ | θ)
Maximise log-likelihood (summing logs avoids numerical underflow with tiny probabilities)
Gaussian: p(x|μ,σ) = (1/√2πσ²) · exp(−(x−μ)²/2σ²)
MLE on Gaussian noise assumption → MSE loss. MLE on Bernoulli → Cross-Entropy loss.
// Gaussian distribution — adjust mean and variance
Mean μ0.0
Std σ1.0
Key insight: Training with MSE loss = assuming your errors are Gaussian distributed. Training with Cross-Entropy = assuming Bernoulli/Categorical outputs. The loss function encodes your distributional assumption.
13 — Core Math

KL Divergence

Information Theory

// Measuring how different two probability distributions are

KL Divergence measures how much information is lost when using distribution Q to approximate P. It is not symmetric — KL(P||Q) ≠ KL(Q||P).

KL(P||Q) = Σ P(x) · log(P(x)/Q(x))
Discrete case  |  Always ≥ 0  |  = 0 only when P = Q exactly
KL(P||Q) = ∫ p(x) · log(p(x)/q(x)) dx
Continuous case  |  used in VAE loss, RLHF, variational inference
Cross-Entropy H(P,Q) = H(P) + KL(P||Q)
CE = entropy of P + extra bits needed because we use Q. Minimising CE ≡ minimising KL.

Bayes' Theorem

Probability

// Updating beliefs with evidence — the foundation of probabilistic ML

Bayes' Theorem is the mathematical foundation of reasoning under uncertainty. It tells us how to update a prior belief when we observe new evidence.

P(A|B) = P(B|A) · P(A) / P(B)
P(A|B) = posterior  |  P(B|A) = likelihood  |  P(A) = prior  |  P(B) = evidence
// Medical test — posterior probability after positive result
Prior P(disease) %1.0%
Sensitivity %95%
Posterior (test+)—
Base rate fallacy: 1% disease prevalence + 95% accurate test = only ~16% chance you're actually sick after a positive. Low priors dominate!

Evaluation

// Reliable model evaluation by rotating which data is used for testing

A single train/test split gives a noisy estimate of generalisation. k-fold cross-validation averages performance over k splits for a more reliable estimate.

CV Score = (1/k) · Σᵢ score(model_i, fold_i)
Train k models, each evaluated on a different held-out fold — no data is wasted

Evaluation

// How to actually measure if your model is good

Precision = TP/(TP+FP)    Recall = TP/(TP+FN)
Precision = of predicted positives, how many are real?  |  Recall = of real positives, how many found?
F1 = 2·(Precision·Recall)/(Precision+Recall)

CNN — Convolutions

Architecture

// Exploiting spatial structure with shared local filters

Convolutional layers apply a small learnable filter across the entire input. The same filter detects the same feature regardless of where it appears — translational equivariance.

(f * g)[i,j] = Σₘ Σₙ f[m,n] · g[i−m, j−n]
f = filter/kernel  |  g = input  |  slide filter over input, compute dot product at each position
Output size = ⌊(W − K + 2P)/S⌋ + 1
W = input width  |  K = kernel size  |  P = padding  |  S = stride
// Convolution operation — kernel sliding over input
Kernel Size3×3
Stride1

self.pool  = nn.MaxPool2d(kernel_size=2, stride=2)
self.conv2 = nn.Conv2d(64, 128, kernel_size=3, padding=1)

Embeddings & Word2Vec

Representation

// Mapping discrete tokens to continuous vector spaces

An embedding maps each discrete token to a dense vector where semantic similarity = geometric proximity. Word2Vec learns these by predicting context words.

Skip-gram: max Σ Σ log P(wₒ | wᵢ)
Predict surrounding words from centre word — forces similar words to nearby vectors
king − man + woman ≈ queen
Learned embeddings encode analogical relationships as linear vector arithmetic
// 2D embedding space — semantic clusters
# PyTorch embedding layer
self.embed = nn.Embedding(vocab_size=50000, embedding_dim=256)
x = self.embed(token_ids)  # (batch, seq) → (batch, seq, 256)
Modern embeddings: Word2Vec is the conceptual ancestor. Today BERT/GPT produce contextual embeddings — the same word gets different vectors depending on context.

Attention Mechanism

Architecture

// Selectively focusing on relevant parts of the input

Each token asks: "which other tokens are most relevant to me?" The answer is a weighted sum of values, with weights from dot-product similarity between queries and keys.

Attention(Q,K,V) = softmax(QKᵀ / √d) · V
Q = Query  |  K = Key  |  V = Value  |  √d = scaling to prevent saturation

Click cells to strengthen
Multi-Head Attention: Run h attention functions in parallel, each with different learned projections. Allows attending to different aspects simultaneously: syntax, semantics, coreference.
20 — Deep Learning

Transformer Architecture

Architecture

// The architecture behind BERT, GPT, and all modern LLMs

The Transformer combines multi-head self-attention with position-wise feed-forward networks, residual connections, and layer normalisation. Attention replaces recurrence, enabling full parallelism.

PE(pos, 2i) = sin(pos/10000^(2i/d))
Positional Encoding — sine/cosine waves of different frequencies encode token position
FFN(x) = max(0, xW₁+b₁)W₂+b₂
Position-wise feed-forward: two linear layers with ReLU. Applied identically at each position.
x = LayerNorm(x + Sublayer(x))
Residual + LayerNorm — applied around every sublayer. Enables training very deep networks.
// Transformer block diagram
encoder_layer = nn.TransformerEncoderLayer(
    d_model=512, nhead=8, dim_feedforward=2048,
    dropout=0.1, norm_first=True
)
transformer = nn.TransformerEncoder(encoder_layer, num_layers=6)

RNN — Recurrent Networks

Sequential

// Processing sequences by passing hidden state through time

A Recurrent Neural Network processes sequences step-by-step, maintaining a hidden state h that carries memory of previous inputs. The same weights are reused at every timestep.

hₜ = tanh(Wₕ·hₜ₋₁ + Wₓ·xₜ + b)
hₜ = new hidden state  |  hₜ₋₁ = previous state  |  xₜ = current input
// RNN unrolled through time
Seq length4

self.rnn = nn.RNN(input_size=10, hidden_size=64, num_layers=1, batch_first=True)

Vanishing gradient: Gradients shrink exponentially over many timesteps. LSTM and GRU solve this with gated memory.
22 — Sequence Models

LSTM — Long Short-Term Memory

Sequential

// Gated memory cells that solve the vanishing gradient problem

LSTMs add a cell state c — a "memory highway" — alongside the hidden state. Three gates control what information flows through.

fₜ=σ(Wf·[hₜ₋₁,xₜ])   iₜ=σ(Wi·[hₜ₋₁,xₜ])   oₜ=σ(Wo·[hₜ₋₁,xₜ])
f = forget  |  i = input  |  o = output gates
cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ tanh(Wc·[hₜ₋₁,xₜ])
Cell state update: forget old + write new candidate
hₜ = oₜ ⊙ tanh(cₜ)
Hidden state = filtered cell state
// LSTM gate diagram
self.lstm = nn.LSTM(input_size=10, hidden_size=64, num_layers=2,
                    batch_first=True, dropout=0.2)
out, (hn, cn) = self.lstm(x)
23 — Sequence Models

GRU — Gated Recurrent Unit

Sequential

// LSTM's streamlined sibling — two gates, one state vector

The GRU simplifies LSTM by merging the cell state and hidden state and using only two gates: update and reset. Fewer parameters, comparable performance.

zₜ = σ(Wz·[hₜ₋₁,xₜ])    rₜ = σ(Wr·[hₜ₋₁,xₜ])
z = update gate  |  r = reset gate
h̃ₜ = tanh(W·[rₜ⊙hₜ₋₁, xₜ])
Candidate hidden state — gated by reset
hₜ = (1−zₜ)⊙hₜ₋₁ + zₜ⊙h̃ₜ
Interpolate between old and new state via update gate
// RNN vs GRU vs LSTM — parameter count comparison
self.gru = nn.GRU(input_size=10, hidden_size=64, batch_first=True)
out, hn = self.gru(x)
When to use which: RNN — quick baseline, very short sequences. GRU — best default for most seq tasks. LSTM — complex long-range dependencies. Transformer — when you have lots of data and GPU.

VAE — Variational Autoencoder

Generative

// Learning a structured latent space for generation and interpolation

A VAE encodes inputs to a distribution over latent vectors (not a single point), then decodes samples from that distribution. This forces the latent space to be smooth and continuous.

ELBO = E[log p(x|z)] − KL(q(z|x) || p(z))
ELBO = VAE objective: reconstruction + KL regularization
z = μ + σ·ε    ε ~ N(0,1)
Reparameterization trick: makes sampling differentiable
// VAE architecture — encoder → latent distribution → decoder
class VAE(nn.Module):
    def forward(self, x):
        mu, log_var = self.encode(x)
        z = mu + torch.exp(0.5*log_var) * torch.randn_like(mu)
        return self.decode(z), mu, log_var

def vae_loss(recon_x, x, mu, log_var):
    recon = F.mse_loss(recon_x, x)
    kl    = -0.5 * torch.mean(1 + log_var - mu**2 - log_var.exp())
    return recon + kl

Diffusion Models

Generative

// Generating by learning to reverse a noise process (DDPM, Stable Diffusion)

Diffusion models learn to denoise data. The forward process gradually adds Gaussian noise over T steps until pure noise. A neural network learns to reverse this — generating data from noise.

Forward: q(xₜ|xₜ₋₁) = N(xₜ; √(1−β)·xₜ₋₁, β·I)
Adds small amount of noise β at each step. After T steps, data ≈ pure Gaussian noise.
xₜ = √ᾱₜ·x₀ + √(1−ᾱₜ)·ε    ε~N(0,I)
Closed form for any timestep t. ᾱₜ = product of (1−βᵢ)
L = E[||ε − εθ(xₜ,t)||²]
Training: predict the noise ε that was added.
// Forward diffusion — noise being added over timesteps
Timestep t0
DDPM → DDIM → Latent Diffusion: DDPM was the original (2020). DDIM made sampling 10–50× faster. Latent Diffusion (Stable Diffusion) runs in a compressed latent space — enabling image generation on consumer GPUs.
27 — Generative & Prob.

GANs — Generative Adversarial Networks

Generative

// Two networks competing: generator vs discriminator

GANs pit a Generator G against a Discriminator D in a minimax game. G learns to generate realistic samples; D learns to distinguish real from fake.

min_G max_D V(D,G) = E[log D(x)] + E[log(1−D(G(z)))]
Minimax objective: D maximises, G minimises. At equilibrium: D(x) = 0.5 everywhere.
G: z → x̃    z ~ N(0,I)    D: x → [0,1]
G maps noise to data space  |  D outputs probability of being real
// GAN training dynamics — Generator vs Discriminator loss
# Training loop (simplified)
for real_x in dataloader:
    D_real = discriminator(real_x)
    fake_x = generator(torch.randn(batch, latent_dim))
    D_fake = discriminator(fake_x.detach())
    d_loss = -torch.mean(torch.log(D_real) + torch.log(1-D_fake))
    g_loss = -torch.mean(torch.log(discriminator(generator(z))))
Mode collapse: The biggest GAN failure — G learns to generate only a few convincing samples. Wasserstein GAN (WGAN) and gradient penalty (WGAN-GP) are the standard fixes.
